# 2025-12-09 데이터 소실 문제 - 최종 정리

## 문제 상황
- 당신이 저장한 Test123=123 데이터가 프로그램 재시작 후 사라짐
- RunDatStoreTests()가 실행될 때마다 같은 파일을 덮어씀
- 테스트 파일 경로와 실제 데이터 경로가 혼동됨

## 근본 원인
**테스트 코드와 실제 데이터 저장소가 같은 경로를 사용**

```
프로그램 시작
  ↓
당신: "/ROMDISK/tmp/Demo.dat"에 Test123=123 저장
  ↓
프로그램 재시작
  ↓
RunDatStoreTests() 실행
  ├─ Test 1에서 "/ROMDISK/tmp/Test_Basic.dat" ← 원래는 Demo.dat와 분리
  └─ 테스트 코드가 같은 파일에 덮어쓰기
  ↓
당신의 Test123=123 사라짐 ❌
```

## 적용한 수정사항

### 1️⃣ 테스트 경로 분리
```
실제 데이터:     /ROMDISK/tmp/Demo.dat
테스트 1-6:      /ROMDISK/tmp/Test_DatStore_Test1-6.dat
```

### 2️⃣ Demo 함수 추가
```vb
RunDatStoreDemo()
  ├─ Demo.dat 로드 (기존 데이터 보존)
  └─ 새 데이터 추가 후 저장
  
RunDatStoreTests()
  ├─ 별도 경로 사용 (영향 없음)
  └─ 기능 검증만 수행
```

### 3️⃣ 저장 방식 통일 (Sync)
```
XML: Sync 방식 (Reset 있음 → 파일 데이터로 완전 초기화)
DAT: Sync 방식 (Reset 추가 → 동일하게 통일)
```

### 4️⃣ 테스트 모듈 생성
```
Test_DataPersistence.gpl
  ├─ Phase 1: 초기 데이터 저장
  ├─ Phase 2: 같은 인스턴스에서 로드
  ├─ Phase 3: 새 인스턴스에서 로드 (검증!)
  ├─ Phase 4: 기존 + 새 데이터 저장
  └─ Phase 5: 최종 검증
```
