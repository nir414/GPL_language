## 2) 모듈 상단 의존성(Requires/Used by) 주석 추가

### 목적
- `Project.gpr` 로드 순서/모듈 분리/이동 시 의존 관계를 빠르게 파악하기 위해
  각 모듈 파일 상단에 간단한 의존성 메타를 기록.

### 포맷
- `Requires:` 이 파일이 직접 참조하는 모듈(주로 `Core_*/Net_*/Data_*/Storage_*`)
- `Used by:` 대표적으로 이 파일을 호출/사용하는 상위 모듈

### 비고
- `Used by`는 "대표" 중심이며, 실제 참조 관계는 코드 검색/빌드로 최종 확인.

## 5) Sandbox: Static 로컬 변수 초기화 주의사항

### 현상(재현)
- 파일: `Test_robot_sandbox/Entry_Main.gpl`
- `Static` 로컬 변수에 **선언과 동시에 초기값을 주면**, 호출할 때마다 값이 다시 초기화되는 듯한 동작을 관찰.

관찰 로그 예시:
- `[StaticTest1] tag=i=1, counter=1`
- `[StaticTest2] tag=i=1, counter=1`
- `[StaticTest1] tag=i=2, counter=2`
- `[StaticTest2] tag=i=2, counter=1`  ← 누적되지 않고 1로 유지
- `[StaticTest1] tag=i=3, counter=3`
- `[StaticTest2] tag=i=3, counter=1`

### 결론
- 이 환경에서는 **Static 로컬 변수는 “선언 시 초기화(= 0)”를 피하는 것이 안전**.
- 초기값이 필요하면 “첫 호출 1회만” 초기화하는 가드 패턴을 사용.

### 권장 패턴(1회 초기화 가드)
```vb
Private Sub Example()
  Static inited As Integer
  Static counter As Integer
	
  If inited = 0 Then
    counter = 0
    inited = 1
  End If
	
  counter = counter + 1
End Sub
```
